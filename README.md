# Prompt Archive - 图片提示词收藏

收录优质的 AI 图片生成提示词，按主题分类整理。

## 结构

```
prompt-archive/
├── prompts/              # 正式收录提示词
│   ├── portrait/        # 人物肖像类
│   ├── landscape/       # 风景场景类
│   ├── concept-art/     # 概念设计类
│   ├── character-design/# 角色设计类
│   ├── movie-poster/    # 电影海报类
│   └── other/           # 其他
├── index/
│   ├── INDEX.md         # 总索引（按分类排列）
│   └── REVIEW_INDEX.md  # （已废弃，改用 GitHub Issue）
└── README.md
```

## 工作流

```
定时任务抓取 → GitHub Issue（标质量等级）→ 你审批 → 自动收录到分类
```

### 质量分级

Issue 创建时会自动打上质量标签：

| 标签 | 含义 | 收录优先级 |
|------|------|-----------|
| `质量-优质` | 独特风格/完整参数/强辨识度 | 🔝 最高 |
| `质量-中等` | 可用但较常见/部分参数缺失 | 中 |
| `质量-一般` | 摘要型/泛用型/参考性内容 | 低 |

你审批时可根据实际内容调整标签。

### 审批方式

在 GitHub Issue 页面：
- **通过** → 添加 `approved` 标签，然后关闭 Issue
- **淘汰** → 直接关闭（不添加 approved 标签）

收录后会在 INDEX 中按 `优质 → 中等 → 一般` 排序展示。

定时任务每 **5 小时**自动搜索抓取一次。

## 收录标准

1. ✅ 有**完整或接近完整**的提示词原文
2. ✅ 有**具体帖子/网页链接**（可验证来源）
3. ✅ 有**作者**信息
4. ⚠️ 效果图：有则最好，无则可补充
5. ✅ 风格描述清晰

## 命名格式

```
prompts/分类/名称-YYYYMMDD.md
```

**格式说明**：名称在前，日期在后，精确到年月日。

## 分类说明

- **portrait**: 人物肖像、头像、半身像
- **landscape**: 风景、环境、场景
- **concept-art**: 概念艺术、游戏原画
- **character-design**: 角色设计、NPC、怪物
- **movie-poster**: 电影海报、书籍封面
- **other**: 其他混合风格
